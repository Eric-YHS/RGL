# 场景美化（Look Dev）建议与落地清单

> 目标：在不影响实验任务与信号灯辨识的前提下，让“建筑 / 天空 / 环境”更可信、更有层次、风格统一。

## 总原则（实验优先）

1. **信号灯永远是视觉主角**：建筑与天空要“好看但不抢戏”（降低饱和度与对比度、控制高亮面积）。
2. **保持时间感一致**：建议只选一个时段（清晨 / 正午 / 黄昏），全场景光色一致。
3. **减少高频纹理**：远景建筑的细碎纹理容易闪烁（moiré），尽量用大块分割与简单窗格。
4. **性能可控**：优先用“少量高质量变化 + LOD”，不要无脑堆大量模型与后期。

## 快速增益（优先做）

### 1) 环境光 / 反射（IBL）

- 你项目里已经有 HDRI：`public/hdri/daytime.hdr`、`public/hdri/sunset.hdr`
- 当前已接入做法：HDRI **同时作为** `scene.background`（天空背景）与 `scene.environment`（IBL）。
- 如果你想更“克制/不抢戏”：可以把背景改回程序天空，只保留 `scene.environment`。
- 关键点：用 `PMREMGenerator` 预过滤（IBL 质感会立刻上一个台阶）。

### 2) 建筑“轮廓层次”优先于“贴图细节”

你当前建筑是 `InstancedMesh(BoxGeometry)`，已经很适合性能，但视觉上最欠缺的是“天际线”和“体块变化”。

两条路线：
- **路线 A（推荐，落地快）**：引入低模建筑资产包（GLB/GLTF），用多个“建筑类型”拼成街区天际线。
- **路线 B（坚持程序化）**：把单盒子升级为“裙楼 + 塔楼 + 退台 + 女儿墙/屋顶机房盒子”的组合体（轮廓变化最显著）。

### 3) 窗户发光更自然

你现在的“发光窗户 Plane”已经有效，但更高级/更稳的是：
- 在建筑材质上做 **Emissive Map（黑底白窗）**，再配合轻度 Bloom（如果你后续愿意加后期）。
- 仍然保持“随机亮窗”，但要做成“片区亮/暗”而不是“点状随机”，更像真实楼宇。

## 建筑美化的“结构化打法”（建议按这个顺序）

1. **街区秩序**：沿街两排建筑带（更整齐），后排更高更密（形成天际线），路口附近留空（突出信号灯）。
2. **体块三段式**：底部（店面/入口）更暗更重；中段规律窗格；顶部收口（女儿墙/机房盒子）。
3. **材质统一**：2–3 个主色系（暖灰混凝土 / 蓝灰玻璃 / 少量点缀），整体去饱和。
4. **远近分层**：近景用 Kenney 高细节建筑；远景用 `low-detail-building-*` 或简化体块。

## 更写实路线（HDRI + PBR + 后期）的落地建议

> 这条路线会让“塑料感”明显下降，但也更需要你控制亮度与后期强度，避免抢走信号灯注意力。

### A. HDRI（IBL）接入

- 推荐：HDRI 用作 `scene.environment`（反射/环境光），背景仍保留你当前的程序天空（更可控）。
- 如果你追求更写实：也可以把 HDRI 设为 `scene.background`（但注意与雾色/色调映射的统一）。本项目当前已采用该方式。

### B. PBR 材质接入（地面/路面/建筑）

- 直接使用 `MeshStandardMaterial` 的组合：
  - `map`（Diffuse/Albedo，**sRGB**）
  - `normalMap`（**Linear**）
  - `roughnessMap`（**Linear**）
  - `aoMap`（可选，**Linear**；需要 `uv2`）
- 建议先把 **路面与人行道** 换成 PBR（收益最大），再考虑建筑立面。

### C. 阴影（强烈建议，但要做“轻量化”）

- 开启软阴影会极大提升真实感：树、路灯、信号灯与人物的接地关系会更扎实。
- 但要控制 shadow map 分辨率与范围（实验场景更重要的是稳定性与帧率）。

### D. 后期（从“微量”开始）

建议最小组合（按顺序）：
1. `RenderPass`
2. SSAO/SAO（二选一，强度低）
3. `UnrealBloomPass`（强度低、阈值偏高，主要让信号灯与窗户更“有光”）
4. 抗锯齿（SMAA/FXAA，视设备而定）

不建议（除非你非常克制）：强 DOF、强色差、重胶片颗粒，会影响实验辨识与注意力。

## 天空 / 氛围（不抢戏的升级点）

- **统一雾色**：让雾的颜色贴近天空底色（你现在就是这么做的），能显著提升空间深度。
- **云层运动更慢**：只要有“轻微流动”就够了，避免参与者分心。

## 人物（可选项）

- 如果你愿意引入外部人物：Mixamo 骨骼动画确实是代入感飞跃，但需要你手动下载/转换。
- 如果不引入模型：保持当前简化小人也没问题，重点是别让人物成为视觉焦点（低饱和、动作幅度克制）。
